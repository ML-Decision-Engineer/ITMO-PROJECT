# Бенчмаркинг продукта: внутренний ML-сервис прогнозирования дефектов продукции

## 1. Целевая аудитория и проблема

Проект является внутренним ML-сервисом для металлургического производства. Поэтому под конкурентами понимаются не публичные SaaS-продукты, а текущие альтернативы и отраслевые бенчмарки.

Внутренний заказчик и производственная площадка не раскрываются по требованиям руководства. Для сравнения используются только открытые публичные источники.

**Целевая аудитория продукта:**

* отдел качества;
* технологи;
* руководство производства.

**Проблема:**

Фактический результат контроля качества становится известен только через 10-14 дней после лабораторной проверки. Из-за этого производство узнаёт о дефектах поздно, когда часть решений по продукции уже могла быть принята.

**Будущий продукт:**

Внутренний ML-сервис, который заранее прогнозирует риск дефекта по продукции текущего дня и рассчитывает ожидаемый дневной процент дефекта до получения лабораторного факта.

---

## 2. Типы конкурентов / бенчмарков

Для анализа выбраны три типа аналогов.

### 2.1. Прямой аналог

**Текущий ручной и лабораторный процесс контроля качества.**

Он решает ту же задачу для той же аудитории: подтверждает факт качества продукции. Его сильная сторона — официальность и доверие к результату. Слабая сторона — задержка результата на 10-14 дней.

### 2.2. Косвенные аналоги

**НЛМК и ММК: цифровой контроль качества.**

НЛМК используется как пример цифровизации лабораторного контроля через LIMS.
ММК используется как пример машинного зрения для выявления дефектов поверхности металлопроката.

Эти решения важны как бенчмарки цифровизации качества, но в открытых источниках по ним не найдены сопоставимые метрики именно для нашей predictive-задачи: прогноз дефекта по производственным данным, дневной процент дефекта, Precision / Recall / Recall@Top-K.

### 2.3. Отраслевые ML-бенчмарки

**SSAB/VTT и ArcelorMittal Brazil.**

SSAB/VTT выбран как пример раннего ML-прогноза дефектов стали и риск-ранжирования продукции.

ArcelorMittal Brazil выбран как пример AI-инспекции качества рулонов на промышленном потоке и оценки бизнес-эффекта.

## 4. Значимые параметры продукта

На основе анализа важны следующие параметры:

1. **Скорость получения результата** — прогноз должен появляться до лабораторного факта.
2. **Precision** — основная операционная метрика, так как ложные тревоги создают лишнюю проверку, карантин и нагрузку на поток.
3. **Recall** — показывает, какую долю дефектов модель способна найти.
4. **Recall@Top-K** — показывает, сколько дефектов попадает в ограниченный список наиболее рискованной продукции.
5. **Ошибка дневного процента дефекта** — бизнес-метрика, связывающая модель с производственным управлением.
6. **Explainability** — технологи должны понимать, почему продукция попала в риск-зону.
7. **Промышленный масштаб** — решение должно работать на десятках тысяч рулонов и листов.
8. **Экономический эффект** — снижение брака, rework, карантинов, возвратов и позднего выявления дефектов.

## 5. Значения бенчмарков и источники

### 5.1. НЛМК

В открытых источниках НЛМК описывает внедрение LIMS — лабораторной информационной системы для контроля качества металлопродукции. Система автоматизирует входной контроль сырья, технологический контроль и испытания готовой продукции.

Найденные значения:

* более 500 методик измерений;
* 250 форм отчётов;
* 63 лабораторные установки;
* более 400 активных пользователей.

Для нашей задачи это benchmark по цифровому контуру качества, но не по ML-метрикам.

### 5.2. ММК

ММК описывает внедрение машинного зрения для выявления и классификации дефектов поверхности холоднокатаного металлопроката в производственном потоке.

Найденное значение:

* точность распознавания дефектов — не менее 95%.

Для нашей задачи это benchmark по автоматическому контролю дефектов, но не прямой benchmark для прогноза дневного процента дефекта.

### 5.3. SSAB/VTT

SSAB/VTT описывает ML-систему explainable steel quality prediction на основе Gradient Boosting Decision Trees. Модель использовалась для раннего выявления риска поверхностных дефектов стали.

Найденные значения:

* 296 производственных параметров;
* 89 входных признаков;
* модель находила более 50% дефектной продукции;
* при этом около 10% продукции помечалось как потенциально рискованная;
* использовалась объяснимость через SHAP.

Для нашей задачи это главный benchmark по Recall@Top-K: модель должна концентрировать значимую долю дефектов в небольшой top-risk группе.

Для нашего объёма в 109 000 рулонов и листов top-10% — это примерно 10 900 единиц продукции. Такой список может использоваться как зона приоритетной проверки.

### 5.4. ArcelorMittal Brazil

ArcelorMittal Brazil описывает Standard Coil — AI-решение для инспекции качества стальных рулонов.

Найденные значения:

* проверено более 43 000 рулонов;
* достигнуто 89% assertiveness;
* индекс качества IQ = 99.70%;
* rework снижен на 30%;
* потенциальный экономический эффект около R$1 млн в год.

Для нашей задачи это benchmark по промышленному масштабу и бизнес-эффекту.

## 6. Минимальные требования к нашему продукту

### Функциональные требования

* ежедневный прогноз риска дефекта;
* расчёт дневного процента дефекта;
* top-risk список продукции;
* сравнение прогноза с лабораторным фактом;
* отчёт для отдела качества, технологов и руководства;
* объяснение факторов риска.

### ML-бенчмарки

* качество выше baseline;
* PR-AUC выше базовой частоты дефекта;
* Precision около 65%;
* Recall не ниже 40%;
* Recall@Top-K для оценки риск-ранжирования;
* ошибка дневного процента не более 5 п.п.;
* ошибка недельного процента не более 2 п.п.

### Продуктовые бенчмарки

* прогноз доступен до лабораторного факта;
* результат понятен производству;
* сервис помогает выбрать продукцию для дополнительной проверки;
* решение можно расширить с одного свойства на несколько.

## 7. Модель ценности

Так как продукт внутренний, внешняя монетизация не применяется.

Ценность продукта считается через внутренний ROI:

* снижение брака;
* снижение переплавки;
* снижение карантинов;
* снижение возвратов;
* сокращение позднего выявления проблем;
* более точное планирование контроля качества.

Главная метрика ценности — снижение производственных потерь и стоимости плохого качества.

## 8. Итог

Главный стандарт продукта: это не просто ML-модель, а инструмент раннего управления качеством.

**Минимальный целевой формат:**

внутренний ML-сервис, который по данным текущего дня заранее показывает риск дефекта, рассчитывает ожидаемый дневной процент дефекта и выделяет продукцию для дополнительного контроля до получения лабораторного факта.

**Ключевое конкурентное преимущество:**

узкая адаптация под конкретный производственный процесс, задержанный лабораторный таргет и дневную бизнес-метрику качества.
