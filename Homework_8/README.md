**КРАТКОЕ ОПИСАНИЕ**

В предыдущих заданиях я сформулировал продуктовую задачу, подготовил данные, обучил и улучшил модель прогнозирования дефекта.

В ДЗ №8 я планирую не переобучать модель, а упаковать готовое решение в минимальный сервис. Пользователь сможет зарегистрироваться, войти в рабочий кабинет, получить прогноз для одной единицы продукции и загрузить данные за производственный день.

Для дневного расчёта сервис выдаст вероятность для каждой строки, ожидаемый процент дефекта и top-risk список. История расчётов будет храниться в PostgreSQL. REST API сделаю на FastAPI, интерфейс — на Streamlit, всё запущу через Docker Compose. Количество воркеров модели будет настраиваться через переменную окружения.

В репозиторий положу только обезличенные демонстрационные данные, модель, тесты и подробный README.


**Вопросы**

1. Правильно ли, что для макс балла достаточно связки FastAPI + Streamlit + PostgreSQL + Docker Compose?
2. Можно ли для одиночного прогноза использовать выбор готовой демонстрационной строки из производственного датасета или загрузку CSV из одной строки, чтобы не создавать форму на десятки признаков?
3. Правильно ли рассчитывать дневной прогноз как среднее калиброванных вероятностей всех строк загруженного дня и отдельно выводить top-risk список?
4. Обязательны ли регистрация и авторизация или это дополнительная функция, не влияющая на оценку?
5. Достаточно ли для пункта о масштабировании задавать число процессов:
uvicorn app.main:app --workers 2 или требуется отдельный сервис очередей и воркеров?

6. Нужно ли хранить в БД все входные признаки каждой строки или достаточно истории запусков и результатов прогнозирования?
7. Можно ли использовать только небольшой обезличенный демонстрационный датасет, не публикуя реальные производственные данные?
8. Нужно ли разворачивать сервис во внешнем облаке или достаточно локального запуска через Docker Compose?
9. Какие именно критические части обязательно покрыть тестами для получения полных 2 баллов?
10. Достаточно ли в README показать архитектуру, команды запуска, API, тесты, масштабирование и сценарий демонстрации?
